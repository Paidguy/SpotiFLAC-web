# Documentation

This directory contains all project documentation organized into subdirectories.

## Directory Structure

- **`development/`** - Development-related documentation, audits, migration reports, and phase analyses
  - Contains historical documentation about the codebase development process
  - Bug fix lists and audit reports
  - Migration status and phase completion reports

## Main Documentation

For user-facing documentation, please see the main [README.md](../README.md) in the project root.

## Contributing

When adding new documentation:
- Place user-facing guides in the main README
- Place development/technical documentation in `development/`
- Keep filenames descriptive and use kebab-case or SCREAMING_SNAKE_CASE
